# Future Features

This file tracks deferred product work that should stay visible after `0.0.1`.

## Setup Sharing

Status: deferred after `0.0.1`

Desired behavior:
- publish a shareable setup bundle to an operator-chosen location
- import a setup bundle from an operator-provided URL
- optionally scaffold a repo for operator-controlled packages or policies

Not included in `0.0.x`:
- repo creation UX
- gist publishing UX
- remote setup import UX

## Moorline Package Registry

Status: deferred after `0.0.x`

Desired behavior:
- first-party hosted registry for Moorline installables and add-ons
- package discovery without manual archive URLs
- publisher flow for third-party authors
- central checksum and version metadata

Not included in `0.0.x`:
- registry server
- publish workflow
- surface or account management
